## 🎯 このシリーズの目的

エージェントを「フロー」や「タスク分解」ではなく、
**「文脈」「実行」「時間」の三軸**で整理する。

構造が整えば、多くの問題は解消される。

## 👥 対象読者

- AIエージェントを設計するエンジニア
- LLMを使った業務自動化を考えるコンサルタント
- データ分析や意思決定支援の仕組みを作る人
- 「自律型のAI」が本当に必要か迷っている人
- AIをどう構造化すれば安定するのかを知りたい人
